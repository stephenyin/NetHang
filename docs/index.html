Overview

NetHang is a network simulation tool that allows you to create and manage network paths between LAN and WAN interfaces. It helps you simulate various network conditions and monitor their effects on traffic.

Paths

A path represents a connection between your LAN and WAN interfaces. Each path can be configured with:

  • Protocol (TCP, UDP, or IP)
  • Source and destination IP addresses
  • Port numbers (for TCP and UDP)
  • Simulation settings
Note: For IP protocol paths, port numbers are not required.

Protocols

TCP

Transmission Control Protocol - Provides reliable, ordered, and error-checked delivery of data.

UDP

User Datagram Protocol - Provides fast but unreliable delivery of data.

IP

Internet Protocol - Basic network layer protocol without port numbers.

Simulation

You can simulate network conditions in two ways:

Models

Pre-configured network models that simulate common network conditions.

Custom Settings

Configure specific network parameters:

  • Bandwidth (Kbps)
  • Queue Depth (Packets)
  • Delay (ms)
  • Packet Loss (%)
  • Jitter (ms)

Monitoring

The application provides real-time monitoring of network conditions through three charts:

Throughput

Shows the bit rate for both uplink and downlink traffic.

Queuing

Displays the number of packets in the queue.

Loss

Shows the packet loss rate over time.
